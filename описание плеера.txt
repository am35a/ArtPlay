Собери SPA-аудиоплеер на чистом Svelte (JS, без TypeScript) в текущем проекте.
UI-слой делай на S4 (S4 уже подключен в index.html). Без Winamp-стиля, без сторонних UI-библиотек.

Источник данных:
- public/library/index.json

Важно по данным:
1) Обнови генератор манифеста и сам манифест:
- добавь поле фото автора в manifest:
  artist.photo (string | null)
- если у автора нет фото, в UI показывай пустой плейсхолдер/пустую картинку с alt = имя автора
- «исполнитель» = artist.name (это имя первой папки в library)
- порядок треков на Главной: как в манифесте (не переупорядочивать дополнительно)

2) Состояние плеера:
- один глобальный Svelte store:
  currentTrack, queue, queueContext, currentTime, duration, isPlaying, volume, muted, shuffle, repeatMode(off|one|all), selectedArtist, selectedAlbum, activeScreen, prevScreenStack
- Мини-плеер и Экран 5 должны быть синхронизированы через один store

3) Навигация:
SPA с 5 экранами:
- Экран 1: Главная
- Экран 2: Авторы
- Экран 3: Автор
- Экран 4: Альбом
- Экран 5: Сейчас играет

Глобальные элементы (на экранах 1–4):
- Шапка: логотип ArtPlay + поиск
- Мини-плеер: обложка, трек+исполнитель (бегущая строка при переполнении), Prev/Play-Pause/Next, тонкий прогресс
- Нижний тулбар (4 пункта): Главная, Авторы, Автор, Альбом

Для Экрана 5:
- НЕ показывать шапку, мини-плеер и тулбар
- наверху кнопка Назад и заголовок
- Назад возвращает на предыдущий экран (история переходов)

4) Поведение кнопок тулбара:
- «Автор» и «Альбом» неактивны, пока не выбран автор/альбом
- при неактивном состоянии клика нет

5) Поиск:
- работает на экранах 1–4, на экране 5 поиска нет
- ищет одновременно по трекам, авторам, альбомам
- фильтрация без перезагрузки

6) Содержимое экранов:
Экран 1 — Главная:
- список всех треков в порядке манифеста
- строка: обложка | название трека | исполнитель | длительность
- активный трек подсвечен success

Экран 2 — Авторы:
- сетка карточек авторов
- карточка: фото автора + имя + список альбомов (превью обложки + количество треков)
- клик по автору -> Экран 3 и активирует кнопку «Автор»

Экран 3 — Автор:
- список альбомов автора
- каждый альбом раскрыт и показывает треки
- строка трека: обложка | название | исполнитель | длительность
- клик по заголовку альбома -> Экран 4 и активирует кнопку «Альбом»

Экран 4 — Альбом:
- список треков альбома
- строка: обложка | название | исполнитель | длительность

Экран 5 — Сейчас играет:
- большая круглая обложка
- название трека + исполнитель
- текущее время / общая длительность
- визуализатор Web Audio API:
  - режимы: Spectrum / Oscilloscope
  - переключатель режима
- controls:
  - прогресс-бар (обычный S4-стиль)
  - Prev | Rewind | Play/Pause | Fast Forward | Next
  - Rewind/Fast Forward шаг = 10 секунд
  - Shuffle (on/off), Repeat (off -> one -> all)

7) Логика очереди и переходов:
- queue зависит от экрана, с которого запущен трек:
  - с Экрана 1: все треки
  - с Экрана 4: треки альбома
  - с Экрана 3: треки выбранного автора (в порядке экрана)
  - с Экрана 2: при старте из карточки автора — треки автора
- shuffle=on: следующий трек случайный в пределах текущего queue
- repeat:
  - off: стоп в конце queue
  - one: повтор текущего трека
  - all: цикл по queue

8) Persistence (localStorage):
сохранять и восстанавливать:
- currentTrack id
- currentTime
- volume
- muted
- shuffle
- repeatMode
- lastQueueContext
- activeScreen
- selectedArtist / selectedAlbum (если применимо)

9) Технически:
- раздели код на понятные компоненты и store-модули
- не добавляй TypeScript
- минимально необходимый CSS, опирайся на S4
- избегай неиспользуемых файлов/кода
- после изменений обнови манифест и проверь, что сборка проходит

10) Результат:
- рабочий UI всех 5 экранов
- рабочее воспроизведение и синхронизация mini-player <-> now-playing
- поиск
- визуализатор
- сохранение/восстановление состояния
